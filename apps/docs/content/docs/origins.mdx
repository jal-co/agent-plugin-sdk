`ap-sdk` was created by [Sahaj Jain](https://github.com/jnsahaj) as
[**agent-plugin-sdk**](https://github.com/jnsahaj/agent-plugin-sdk). The
ai-sdk-style architecture at its core — a portable
plugin definition, a declarative capability map, and structured warnings instead
of silent failures — is his, along with the original skills, commands,
subagents, hooks, MCP, and shared-tool support across Claude Code, Codex, Pi,
and OpenCode.

## What this project adds

This is a clean continuation of that work, with Sahaj's blessing — not a fork.
Building on his foundation, it adds:

- **Four more harnesses** — Gemini CLI, GitHub Copilot, Cursor, and Windsurf.
- **A runtime harness registry** and an `ap-sdk add-harness` scaffold, so the
  list of supported agents keeps growing.
- **The monorepo and this documentation site.**

The design is still his; we are standing on it.

## Credit

Sahaj is credited in the package metadata, the `LICENSE`, and the README. If
`ap-sdk` is useful to you, star the [original repository](https://github.com/jnsahaj/agent-plugin-sdk)
and a nod to [@jnsahaj](https://github.com/jnsahaj) is always appreciated.
